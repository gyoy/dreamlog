# AI_GUIDE.md

이 프로젝트는 실제 iOS 앱 출시를 목표로 하는 앱 프로젝트다.
단순 데모나 임시 구현이 아니라, 유지보수 가능하고 App Store 제출 가능한 품질을 목표로 한다.

## 1. 최우선 원칙

* 사용자가 요청한 범위 밖의 코드는 임의로 수정하지 않는다.
* 간단한 수정 요청을 받았다고 해서 전체 구조를 뒤엎지 않는다.
* 많은 파일을 수정해야 하거나 구조 변경이 필요한 경우, 반드시 먼저 사용자에게 변경 계획을 설명하고 확인을 받는다.
* 요청이 명확하지 않으면 추론해서 바로 실행하지 말고, 먼저 사용자의 의도를 어떻게 이해했는지 정리한다.
* 기존에 정상 작동하던 기능을 망가뜨리지 않는 것을 최우선으로 한다.
* 작업 전 항상 관련 파일을 먼저 읽고 현재 구조를 파악한다.
* 작업 후에는 변경한 파일, 수정 내용, 잠재적 문제, 다음 작업을 요약한다.

## 2. 코드 수정 규칙

* 하나의 파일에 모든 코드를 몰아넣지 않는다.
* 기능별로 파일과 컴포넌트를 분리한다.
* 화면, 컴포넌트, 훅, 유틸, 서비스, 타입은 가능한 한 역할별로 분리한다.
* 중복 코드는 공통 컴포넌트나 유틸로 정리한다.
* 임시 코드, 테스트용 하드코딩, 불필요한 콘솔 로그는 남기지 않는다.
* 기존 파일 구조와 네이밍 규칙을 우선적으로 따른다.
* 새로운 폴더나 구조가 필요하면 먼저 이유를 설명한다.
* 전체 리팩토링은 명시적으로 요청받았을 때만 진행한다.

## 3. 디자인 구현 규칙

* Figma, SVG, PNG 등 제공된 디자인 에셋을 최우선 기준으로 삼는다.
* 디자인을 임의로 재해석하거나 새롭게 꾸미지 않는다.
* 위치, 크기, 여백, 색상, 그림자, 라운드 값은 제공된 수치를 우선한다.
* 텍스트가 아직 확정되지 않은 영역은 임의 문구를 넣지 말고 빈 상태 또는 placeholder로 처리한다.
* iOS Safe Area를 반드시 준수한다.
* 화면 크기가 달라져도 요소가 깨지지 않도록 반응형으로 구현한다.
* 기본 배경, 버튼, 카드, 탭바 등 공통 UI는 재사용 가능한 컴포넌트로 만든다.

## 4. 기능 구현 규칙

* 큰 기능은 작은 단위로 나누어 순차적으로 구현한다.
* 한 번에 여러 기능을 동시에 구현하지 않는다.
* 먼저 P0 핵심 기능을 구현하고, 이후 P1, P2 기능으로 확장한다.
* 각 기능은 로딩 상태, 빈 상태, 오류 상태를 고려한다.
* 데이터 저장, 수정, 삭제 기능은 예외 상황까지 함께 처리한다.
* 사용자 데이터가 사라질 수 있는 작업에는 확인 절차를 둔다.
* 권한 요청은 필요한 시점에만 수행한다.
* iOS 권한 문구는 실제 기능과 일치해야 한다.

## 5. 병렬 작업 규칙

이 프로젝트는 두 개의 AI 작업 창을 병렬로 사용할 수 있다.

### A 창: 코드 수정 담당

* 실제 코드 수정과 구현을 담당한다.
* 요청받은 범위 안에서만 파일을 수정한다.
* 작업 전 수정 대상 파일을 먼저 확인한다.
* 작업 후 수정 파일 목록과 변경 내용을 요약한다.

### B 창: 검토 담당

* 코드를 직접 수정하지 않고 읽기와 분석을 담당한다.
* 현재 구조, 잠재 버그, 기획 누락, 로직 충돌 가능성을 점검한다.
* A 창이 작업 중인 영역과 충돌하지 않도록 조언한다.
* 필요한 경우 사용자에게 점검 결과와 개선 제안을 제공한다.

## 6. 병렬 작업 시 주의사항

* 두 창이 동시에 같은 파일을 수정하지 않는다.
* A 창은 코드 변경 담당, B 창은 분석 담당으로 역할을 분리한다.
* B 창은 코드 수정이 필요하다고 판단해도 직접 수정하지 말고 사용자에게 제안만 한다.
* A 창은 B 창의 검토 내용을 반영할 때, 변경 범위를 먼저 정리한다.
* 병렬 작업 중 충돌 가능성이 있으면 즉시 작업을 멈추고 사용자에게 보고한다.

## 8. 금지 사항

* 사용자 확인 없이 전체 구조를 변경하지 않는다.
* 사용자 확인 없이 디자인을 새로 만들지 않는다.
* 사용자 확인 없이 라이브러리를 추가하지 않는다.
* 사용자 확인 없이 파일을 대량 삭제하지 않는다.
* 임시 해결책을 최종 구현처럼 처리하지 않는다.
* 빌드가 깨진 상태로 작업을 끝내지 않는다.
* 모르는 부분을 확신하는 척하지 않는다.
