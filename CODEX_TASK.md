# 꿈 통계 화면 (StatsScreen) 리빌드 태스크

## 요청 사항
통계 화면(StatsScreen)을 스크롤 가능하며 가변성이 높은 최상급의 퀄리티로 리빌드해 주세요.

### 상세 요구 사항
1. **달력의 동적 기능 (가변성)**:
   - 이전/다음 월 이동 기능 및 오늘 날짜 바로가기 기능.
   - 각 날짜 아래에 저장된 꿈 기록의 이모지(모드별: 🪐 행성계, 👑 별자리) 표시.
   - 날짜를 클릭하면 해당 날짜의 꿈 기록 리스트만 필터링되어 노출되거나 토글되어야 합니다.
2. **검색 및 필터 기능 (가변성)**:
   - 검색창을 통해 꿈 제목/내용을 검색할 수 있어야 합니다.
   - 필터 드롭다운을 통해 특정 모드(행성계/별자리), 즐겨찾기(⭐), 또는 감정(행복, 평온, 신기함, 두려움, 슬픔)별로 필터링할 수 있어야 합니다.
3. **태그 필 필터링 (가변성)**:
   - 꿈 기록들에 사용된 키워드 태그들을 가로 스크롤 형태로 노출하고, 클릭 시 해당 태그가 포함된 꿈들만 필터링(다중 선택 시 교집합)되어야 합니다.
4. **글꼴 및 디자인 시스템 일관성**:
   - 폰트는 Pretendard를 사용합니다.
   - 보라색 메인 테마와 부드러운 그림자(shadow)를 적용해 디자인 퀄리티를 극대화해 주세요.
5. **목록 스크롤 및 상세 모달**:
   - 스크롤을 통해 필터링된 꿈 기록 전체를 볼 수 있어야 하며, 꿈 카드를 클릭하면 상세 정보를 모달 형태로 조회할 수 있어야 합니다.
   - 상세 모달에서는 삭제 기능과 함께 수정하기 진입 버튼이 있어야 합니다.
6. **빌드 검증**:
   - TypeScript 컴파일 에러가 없어야 합니다 (`npx tsc --noEmit`이 성공해야 함).

## 현재 진행 상태
- `src/screens/StatsScreen.tsx` 및 `src/types/record.ts`에 해당 기능들이 1차 구현되었습니다.
- `npx tsc --noEmit`을 통과하여 컴파일 오류가 없는 상태입니다.
- 이 문서를 기반으로 최종 검증을 완료하고 부족한 연동 부분이 있다면 코드를 고도화해 주세요.
